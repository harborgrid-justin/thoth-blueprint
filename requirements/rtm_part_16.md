# Requirements Traceability Matrix - Part 16
**Subject:** Grading Tutorials (Chapter 11) + Supplemental Feature Line Grading (grading-to-rtm.txt)
**Coverage:** Grading Groups, Target Criteria, Infills, Feature Lines (Creation, Editing, Reference Tools), Complex Footprints

| Req ID | Tutorial Section / Reference | Requirement Description | Traceability | Code Mapping |
|---|---|---|---|---|
| REQ-16-001 | Grading > Grading Groups | The system shall organize grading entities into Grading Groups that support automatic target surface generation. | Trace-to-Spec-v1 | Not implemented |
| REQ-16-002 | Grading > Grading Criteria | The system shall support creating grading criteria with configurable target types (Surface, Distance, Elevation, Relative Elevation) and grading methods (slope, grade). | Trace-to-Spec-v1 | Not implemented |
| REQ-16-003 | Grading > Grading Styles | The system shall support defining grading styles controlling slope pattern display, daylight line color, and visibility. | Trace-to-Spec-v1 | Not implemented |
| REQ-16-004 | Grading > Criteria Sets | The system shall allow organizing multiple grading criteria into Criteria Sets for reuse across projects. | Trace-to-Spec-v1 | Not implemented |
| REQ-16-005 | Grading > Create Grading | The system shall support creating gradings by selecting a feature line and applying a grading criteria (Cut/Fill slopes to surface). | Trace-to-Spec-v1 | Not implemented |
| REQ-16-006 | Grading > Infills | The system shall allow creating grading infill areas (pads, shoulders) inside closed feature line regions. | Trace-to-Spec-v1 | Supported via closed `GradeRegion` boundaries that grade inside a region boundary in [terrain.ts](../packages/domain/src/terrain.ts#L344) and custom target elevations updated via [GradingSolverDialog.tsx](../apps/web/src/features/survey/GradingSolverDialog.tsx) |
| REQ-16-007 | Grading > Edit Grading | The system shall support editing gradings by modifying feature line geometry and allowing real-time grading surface updates. | Trace-to-Spec-v1 | Live updates to proposed graded surface run dynamically on coordinate/elevation edits in [workspaceStore.ts](../apps/web/src/store/workspaceStore.ts#L80-L91) |
| REQ-16-008 | Grading > Volume Tools | The system shall calculate grading volumes (cut/fill) by comparing grading group surfaces to existing ground. | Trace-to-Spec-v1 | Cuts/fills volume calculations comparing proposed grading regions against existing ground computed in [grading.ts](../packages/domain/src/grading.ts#L61) and earthworks results table rendered in [GradingSolverDialog.tsx](../apps/web/src/features/survey/GradingSolverDialog.tsx#L100) |
| REQ-16-009 | Grading > Complex Footprints | The system shall support grading from complex building footprints with multiple vertices and irregular shapes. | Trace-to-Spec-v1 | `GradeRegion` boundary can be any irregular polygon shape, processed via `pointInPolygon` in [terrain.ts](../packages/domain/src/terrain.ts#L344) |
| REQ-16-010 | Grading > Feature Lines | The system shall support converting 2D polylines into 3D Feature Lines with per-vertex elevation control. | Trace-to-Spec-v1 | Not implemented |
| REQ-16-011 | Grading > Feature Line Elevation Editor | The system shall provide an Elevation Editor for viewing and editing individual vertex elevations, grades, and PI properties on feature lines. | Trace-to-Spec-v1 | Not implemented |
| REQ-16-012 | Grading > Arc Tessellation | The system shall support adjusting Tessellation Spacing to smooth feature line arcs for triangulation accuracy. | Trace-to-Spec-v1 | Not implemented |
| REQ-16-013 | Grading > Crossing Feature Lines | The system shall enforce identical elevation values at points where two feature lines cross (shared vertices and split points). | Trace-to-Spec-v1 | Not implemented |
| REQ-16-014 | Grading > Flatten Grade | The system shall support flattening grades or elevations across a feature line segment. | Trace-to-Spec-v1 | Not implemented |
| REQ-16-015 | Grading > Insert/Delete PI | The system shall support inserting and deleting PIs (point of intersection) on feature lines to refine geometry. | Trace-to-Spec-v1 | Grip editing polygon boundaries (inserting, deleting, moving vertices) is supported in [PlanningCanvas.tsx](../apps/web/src/features/canvas/PlanningCanvas.tsx#L367-L388) |
| REQ-16-016 | Grading > Raise/Lower | The system shall support raising or lowering feature line elevations by a specified amount. | Trace-to-Spec-v1 | Adjusting target elevations of grading regions supported in [PropertiesPanel.tsx](../apps/web/src/features/workspace/PropertiesPanel.tsx#L66-L93) and interactive pad elevation slider/inputs in [GradingSolverDialog.tsx](../apps/web/src/features/survey/GradingSolverDialog.tsx#L80-L90) |
| REQ-16-017 | Grading > Set Elevation From Surface | The system shall support setting feature line elevations from an existing surface. | Trace-to-Spec-v1 | Not implemented |
| REQ-16-018 | Feature Lines > Curves Support | The system shall support feature lines with true arc curves (not approximated chord segments like 3D polylines). | Trace-to-RTM-Grading | Not implemented |
| REQ-16-019 | Feature Lines > Create From Scratch | The system shall provide a Create Feature Lines command prompting for vertex locations and elevations (specific value, from surface, or relative to previous vertex by elevation or slope). | Trace-to-RTM-Grading | Not implemented |
| REQ-16-020 | Feature Lines > Create From Objects | The system shall support creating feature lines from existing AutoCAD objects (polylines, lines, arcs) with options to Erase Existing Entities, Assign Elevations, and Weed Points. | Trace-to-RTM-Grading | Not implemented |
| REQ-16-021 | Feature Lines > Elevation Assignment Options | The system shall support three elevation assignment options when creating from objects: typed value, from grading, or from surface (with optional intermediate grade break point insertion). | Trace-to-RTM-Grading | Not implemented |
| REQ-16-022 | Feature Lines > Relative To Surface | The system shall support setting feature line elevation relative to a surface with a vertical offset, so the feature line dynamically adjusts when the surface changes. | Trace-to-RTM-Grading | Not implemented |
| REQ-16-023 | Feature Lines > Create From Alignment | The system shall support creating feature lines from an alignment + profile, with optional dynamic linking so the feature line updates when alignment or profile geometry changes. | Trace-to-RTM-Grading | Not implemented |
| REQ-16-024 | Feature Lines > Create From Corridor | The system shall support creating regular feature lines from corridor feature lines, with optional dynamic linking to the corridor model. | Trace-to-RTM-Grading | Trace-to-RTM-Grading | 3D feature line coordinates extraction from pavement edges, centerlines, and shoulders computed dynamically at station intervals in [corridor.ts](../packages/domain/src/corridor.ts#L55-L100) |
| REQ-16-025 | Feature Lines > Corridor FL Distinction | The system shall distinguish between corridor feature lines (non-editable, site-independent) and regular feature lines (editable, site-aware, surface-compatible). | Trace-to-RTM-Grading | 3D corridor feature lines are extracted and added as independent, editable spatial elements to the site in [CorridorDesignerDialog.tsx](../apps/web/src/features/survey/CorridorDesignerDialog.tsx#L55-L70) |
| REQ-16-026 | Feature Lines > Break/Trim/Join/Reverse | The system shall support Break, Trim, Join, and Reverse commands for horizontal feature line editing. | Trace-to-RTM-Grading | Not implemented |
| REQ-16-027 | Feature Lines > Edit Curve/Fillet/Fit/Smooth | The system shall support Edit Curve, Fillet, Fit Curve, and Smooth commands for adjusting feature line curve geometry. | Trace-to-RTM-Grading | Not implemented |
| REQ-16-028 | Feature Lines > Weed Vertices | The system shall provide a Weed command to reduce excessive vertices from feature lines based on angle and distance factors. | Trace-to-RTM-Grading | Not implemented |
| REQ-16-029 | Feature Lines > Stepped Offset | The system shall provide a Stepped Offset command to create a new feature line at a specified horizontal distance from an existing one, with elevation options: difference, slope/grade, or specific elevation. | Trace-to-RTM-Grading | Not implemented |

| REQ-16-031 | Feature Lines > Quick Elevation Edit | The system shall provide a Quick Elevation Edit command for graphically editing feature line vertex elevations and slopes directly in the drawing view. | Trace-to-RTM-Grading | Not implemented |
| REQ-16-032 | Feature Lines > Set Grade/Slope Between Points | The system shall support setting a uniform grade/slope between two specified points on a feature line, interpolating across intermediate vertices. | Trace-to-RTM-Grading | Not implemented |
| REQ-16-033 | Feature Lines > Insert High/Low Point | The system shall support inserting a calculated High/Low Elevation Point by specifying start point, end point, and incoming/outgoing grades, automatically computing the grade break location. | Trace-to-RTM-Grading | Not implemented |
