# Requirements Traceability Matrix - Part 80
**Subject:** Curtain Walls Frame & Mullion Details (Chapter 21, Part 2)
**Coverage:** Frame Edge Locations, Mullion Locations, Mullion Indexing, Default Assignments, Overlap Priority Rules

| Req ID | Tutorial Section / Reference | Requirement Description | Traceability | Code Mapping |
|---|---|---|---|---|
| REQ-80-001 | Chapter 21 > Curtain Walls > Edge Location | The system shall support specifying which frame edge (Left, Right, Top, Bottom) uses a specific frame assignment. | Trace-to-Spec-v1 | Not implemented |
| REQ-80-002 | Chapter 21 > Curtain Walls > Mullion Location | The system shall support assigning mullion definitions to Start, Middle, or End locations in a grid. | Trace-to-Spec-v1 | Not implemented |
| REQ-80-003 | Chapter 21 > Curtain Walls > Mullion Index | The system shall support assigning mullion definitions to specific indices (e.g. 1, 2, 3) in a grid. | Trace-to-Spec-v1 | Not implemented |
| REQ-80-004 | Chapter 21 > Curtain Walls > Default Mullion | The system shall automatically apply a default mullion definition to all unassigned mullion edges. | Trace-to-Spec-v1 | Not implemented |
| REQ-80-005 | Chapter 21 > Curtain Walls > Priority | The system shall enforce priority rules where only the last assigned definition applies to a mullion or frame edge. | Trace-to-Spec-v1 | Not implemented |
