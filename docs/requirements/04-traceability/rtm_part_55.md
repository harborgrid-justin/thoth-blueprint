# Requirements Traceability Matrix - Part 55
**Subject:** Stairs (Chapter 27)
**Coverage:** Stair Shapes (Straight/Multi-Landing/Spiral), Turn Types, Vertical Orientation (Up/Down), Riser/Tread Calculation Rules, Stair Terminations

| Req ID | Tutorial Section / Reference | Requirement Description | Traceability | Code Mapping |
|---|---|---|---|---|
| REQ-55-001 | Chapter 27 > Stairs > Shapes | The system shall support creating straight, multi-landing, 1/4-turn, 1/2-turn, and spiral Stair shapes. | Trace-to-Spec-v1 | Not implemented |
| REQ-55-002 | Chapter 27 > Stairs > Turn Types | The system shall support turn configurations: 1/2 landing (flat landing), 1/2 turn (winder treads same direction), 1/4 landing (square landing), and 1/4 turn (winder treads any direction). | Trace-to-Spec-v1 | Not implemented |
| REQ-55-003 | Chapter 27 > Stairs > Orientation | The system shall support vertical orientation properties to specify drawing direction as Up (to floor above) or Down (to floor below). | Trace-to-Spec-v1 | Not implemented |
| REQ-55-004 | Chapter 27 > Stairs > Calculation Rules | The system shall enforce stair tread depth and riser height calculation limits to balance riser count and length. | Trace-to-Spec-v1 | Not implemented |
| REQ-55-005 | Chapter 27 > Stairs > Terminations | The system shall support ending stair runs with a riser, a tread, or a flat landing. | Trace-to-Spec-v1 | Not implemented |
